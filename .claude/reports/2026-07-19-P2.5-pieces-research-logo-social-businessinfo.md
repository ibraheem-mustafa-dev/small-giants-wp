# P2.5 pieces research — LOGO + SOCIAL + BUSINESS-INFO (the ideal, unbiased)

> **The one big cross-cutting lever:** *structured data entered once, rendered contextually.* A single
> Site-Info source (name/phone/email/address/hours), a single social-profile list, and a single logo object
> (all its states) — each edited ONCE and rendered in header + footer + drawer + schema. No competitor theme
> does this cleanly; it's the clearest category-level "exceed". (SGS already part-has it — `source: site-info`
> toggle on social/business-info blocks.)

## LOGO
**MUST:** left-aligned default (NN/g: 6× better home-return, 89% more memorable than right) · link-to-home on
by default · responsive logo = separate desktop/tablet/mobile IMAGE upload (swap the file, not just resize) ·
SVG upload · functional alt text (describe the destination "[Business] home", not "logo" — with an inline
hint) · max-width/height controls (per breakpoint) · sticky-header logo swap (compact mark).
**SHOULD:** transparent-header logo variant (light/dark) · shrink-on-scroll (row+logo dimension animate, not
just an image cut) · dark-mode logo variant · logo+site-title lockup option (togglable) · "sync as site
icon/favicon" toggle (WP core `shouldSyncIcon`).
**NICE:** subtle SVG entrance/hover animation (`prefers-reduced-motion`-gated) · auto-2x raster fallback.
**A11y/SEO:** functional alt mandatory · visible focus (logo is first tab-stop) · logos exempt from contrast SC
but still need alt · `<img>` inside `<a href="/">` near top of DOM = implicit "home" signal.
**EXCEED:** ONE logo *object* attribute (desktop/mobile/sticky/transparent/dark, optional with fallback
chain) in one inspector panel with live fallback preview — beats Kadence/Blocksy's split-across-Customizer-
panels UX. Native SVG-first + automatic favicon derivation.

## SOCIAL ICONS
**MUST:** curated platform set + first-class custom-SVG upload (WP core is painful to extend) · accessible name
per icon auto-generated + editable (WP core omits `aria-label` by default — a citable competitor gap) ·
`rel="noopener noreferrer"` (+optional `nofollow`) auto on external links · open-in-new-tab (default on) ·
size/shape/spacing controls · colour mode brand vs monochrome/theme + hover colour.
**SHOULD:** SINGLE source-of-truth list rendered in header+footer+drawer, each placement independently styled
(no competitor does this — all re-configure per placement) · drag-to-reorder · keyboard-reachable + visible
focus (not hover-only).
**NICE:** explicit "Follow" (profile links, header/footer) vs "Share" (share THIS page, by content) as two
distinct components · per-icon hover micro-interaction (reduced-motion-gated).
**A11y:** icon-only link needs accessible name (verb+platform "Follow us on Instagram"), glyph `aria-hidden` ·
`rel="me"` optional for identity/E-E-A-T.
**EXCEED:** the single site-wide social list independently styled per placement; structural follow/share
separation.

## BUSINESS-INFO / CONTACT (utility-bar content)
**MUST:** click-to-call (`tel:` E.164) · click-to-email (`mailto:` with descriptive text, not a raw address) ·
click-to-map address (Google Maps URL / Place ID) · **single Site-Info source of truth** (name/phone/email/
address/hours edited once → header utility bar + footer + contact page + schema) · schema.org `LocalBusiness`
JSON-LD driven by the SAME fields (Google requires telephone/address/openingHoursSpecification) · responsive
collapse to icon-only on mobile (keep the link + 44px target).
**SHOULD:** opening hours with a live open/closed state ("Open now — closes in 2h") — competitors need a
dedicated plugin for this · utility/top bar as a distinct structural zone above the nav row · multi-location
support (repeat per branch, each schema-tagged).
**NICE:** auto international phone normalisation · pre-filled `mailto:` subject.
**A11y/SEO:** descriptive link text + `aria-label` retained on mobile icon-only collapse · schema generated
server-side from the single source (no plugin data-drift) · semantic `<address>`-adjacent markup.
**EXCEED:** ONE editable Site-Info source powering utility bar + footer + contact page + `LocalBusiness` schema
simultaneously (Kadence/Blocksy/Spectra all re-enter per placement, none auto-generate schema). Native live
open/closed state without a plugin.

## Cross-cutting (for the spec)
- The single biggest meet-and-exceed lever across all three = structured-data-once-rendered-everywhere.
- Responsive = explicit per-breakpoint controls (device-tier), not CSS-only.
- A11y is not optional — WP core's own social block under-delivers `aria-label` by default; close it by default.
