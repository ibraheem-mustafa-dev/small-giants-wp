# Draft вҶ’ SGS clone mapping (layout + responsive CSS)

> **Big finding from the CSS pass (read first):** the structural routing is sound, but the CSS pass exposed a second, larger class of gap вҖ” **built-in-element blocks lack the per-element typography/visual attributes needed to carry the draft's CSS.** `sgs/info-box`, `sgs/testimonial`, `sgs/notice-banner`, `sgs/product-card`, `sgs/option-picker` render text/borders from their own attrs but are missing `headingFontSize`/`quoteFontSize`/`borderRadius`/`padding`/`backgroundColour` etc. Composites that render text via CHILD blocks (heroвҶ’child `sgs/heading`) are fine (the child owns typography, D192). So the work splits cleanly: **converter rules (G1вҖ“G9)** + **DB data (aliases/suffixes/tokens)** + **block-completeness (add the missing per-element attrs to built-in-element blocks).** All three are additive; none is a rearchitecture.
