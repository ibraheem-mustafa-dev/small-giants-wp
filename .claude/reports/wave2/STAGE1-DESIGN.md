# Stage 1 design v2 — universal per-slot CSS dispatch

## The architectural primitive (plain English)
Today the converter decides where a CSS value goes by looking **only at each element's own CSS**, through **four parallel lift functions**, and routes composite interiors through **two per-composite carve-out branches** (`has_scalar_media_attrs`-gated `_route_composite_interior` + `_is_container_mirror_block`).

The universal primitive: **for every DOM node, each CSS declaration (its own + any it inherits) is routed to ONE destination — a child block's attr, the parent composite's per-slot attr, or the block's own scalar attr — by a single DB-driven dispatch that, PER DECLARATION, asks the database "which attribute owns this property for this (block, layer)?" The same dispatch runs at every nesting level, no per-composite branches.** Decision keys, by fork: **structural box CSS** (`contentWidth`/`contentPadding*`/`mediaPadding*`/`gridItem*`) routes NAME-FREE — by **layer detection (CSS signature + position) + `property_suffixes`** (D194); the **CONTENT fork** (emit a child InnerBlock vs a scalar attr) is decided by `canonical_slot` + `role` + `attr_type` together (via `equivalent_block_for`). Both are the DB, NOT Python precedence order. `canonical_slot` is content-fork metadata only — it is **not** the structural-CSS routing key (D194; design-gate `WRAPPER-CSS-ROUTING-DESIGN-GATE.md`).

### Commit 2 — cross-node routing (NET-NEW; council C6) — NO Commit 0a dependency (D194)
New step `_route_interior_css_to_parent_slot(child_node, parent_block, parent_attrs, css_rules)`:
1. Resolve the child's BEM element → its slot (`slots` table) for the CONTENT fork only.
2. **FORK on a SLOT-KEYED predicate FIRST (the FR-22-5 D1 rule) — NOT `equivalent_block_for(block, slot)` (qc-council 2026-06-09 fatal catch).** `equivalent_block_for(block_slug, attr_name)` (`db_lookup.py:1995`) is **attr-keyed** — it queries `WHERE block_slug=? AND attr_name=?`, so passing a slot NAME returns `None` for every call and the fork never fires. Add a new `slot_has_equivalent_block(block_slug, slot_name)` predicate that queries `block_attributes WHERE block_slug=? AND canonical_slot=? AND role IN (<content-bearing roles>)`. This slot-keyed predicate is the CONTENT-fork test ONLY. If TRUE → the CSS attributes to the CHILD block (existing D1 path); FALSE → lift the child's box CSS to the parent's per-slot attr group. **The box-CSS DESTINATION is found NAME-FREE — by layer-prefix + box-property `property_suffixes` (D194), NOT by matching `canonical_slot`.** `canonical_slot` plays no part in selecting the box-CSS destination attr.
3. **EXCLUDE `display`/`grid-template-*` from cross-node** (council MISSING/GAP-3, `convert.py:2791-2799`): grid/display as inline beats `@media` and collapses 2-col grids. Box CSS only (padding/margin/max-width/gap → attrs, never inline grid).
4. **Flag-not-drop (FR-22-21 step 6 + FR-22-2.4):** a slot with no matching parent attr → log an `attribute_gap_candidate` + `unresolved_equivalent_block.log` entry → a gap-candidate to add the attr to the composite. NEVER drop, NEVER `if slug=='sgs/hero'` special-case (council SF-1).
- **Mechanism FR:** this needs **FR-22-5.3** (cross-node interior box-CSS → parent per-slot attr group) written into Spec 22 first — see STAGE0-FRS-AND-GATE.md. **Measure:** hero `contentPadding*` set from `.sgs-hero__content`; trust-bar `contentWidth` from `__inner`.

#### Commit 2 build contract (council-hardened 2026-06-09)
These are the precise REQUIREMENTS the adversarial council (2026-06-09) surfaced for Commit 2. They are NEXT-SESSION build work — recorded here so the build session implements them exactly, not approximations.

- **Per-block attr resolution, NOT prefix concatenation.** Hero uses `contentMaxWidth*` (suffix `MaxWidth`); the other 28 roster blocks use `contentWidth` (suffix `Width`). So a `{layer-prefix} + {property_suffixes}` string-concat CANNOT generate both destination attrs from a single draft `max-width` / `--content-width` signal. The layer→attr resolver MUST be a **per-block lookup**: given `(block_slug, layer, css_property)` → the block's ACTUAL `attr_name` from its mirrored attr set. (FLAG: hero's `contentMaxWidth*` naming DIVERGES from the 28-block `contentWidth` convention — an R-22-9 mirror-divergence. Resolve before/at build: either rename hero's attr to `contentWidth`, or document it as a sanctioned exception.)
- **`slot_has_equivalent_block(block_slug, slot_name)` contract:** query `SELECT 1 FROM block_attributes WHERE block_slug=? AND canonical_slot=? AND role IN (<content-bearing roles>) LIMIT 1`. Returns False for content/layout rows (`role='layout'` is NOT content-bearing). MUST NOT use the attr-keyed `equivalent_block_for` (the qc-council fatal catch — it queries `WHERE block_slug=? AND attr_name=?`, so a slot name returns `None` every call and the fork never fires).
- **CONTENT-WIDTH signature falsification list** — the cases the heuristic must NOT silently miss (each → route to gap-candidate, NEVER guess): `width:min(W,100%)` / `clamp()` with no `max-width`; `margin-inline:auto` longhand; `margin:0 auto` longhand (`margin-left`/`margin-right:auto`); section-root `max-width` with NO inner wrapper; flex-grid (`display:flex;flex-wrap:wrap` + child `width:calc()`); padding-based centring (`padding:0 10vw`).
- **Non-sgs-classed wrapper fold:** scraped drafts have wrappers with NO `sgs-` classes. FR-22-4.1's slug-None fold MUST also cover a non-sgs-classed direct child carrying the content-width signature — else its CSS is lost via the FR-22-11 pass-through.
- **Co-located layers (non-exclusive detection):** one element can be OUTER + CONTENT + GRID simultaneously (e.g. `.sgs-brand`, `.sgs-trust-bar__inner`). Its CSS routes to ALL matching layers' attrs on the SAME container: `max-width`/`--content-width`→`contentWidth` AND `grid-template-columns`→`gridTemplateColumns` AND `gap`→`gap`. Layer detection is non-exclusive and runs on the POST-FOLD tree.
- **SHOULD-FIX (DB hygiene):** `contentLayout` / `contentType` / `contentPosition` / `contentImpact` got `canonical_slot='content'` by stem-collision — they are ENUMS, not content-cap box attrs (inert today via the role gate, but semantically wrong). Add a `_CANONICAL_EXCLUSIONS` entry in `assign-canonical.py` for non-box `content*` stems, OR null them with an explicit non-content canonical_slot.
