# Phase 1.5 /systematic-debugging Round 2 — synthesis + corrected fix shape

## Round 1 was wrong; Round 2 reanchors

Round 1 recommended building 5 bespoke section blocks (`sgs/brand`, `sgs/featured-product`, `sgs/ingredients-section`, `sgs/gift-section`, `sgs/social-proof`). Bean rejected: that violates R-22-9 (universal mechanisms, no per-block hyperfocus, Bean P1 locked 2026-05-25). Round 2 re-dispatches with the correct architectural primitive read end-to-end from Spec 22 + flow + stages docs.

## The architectural primitive (plain English)

The walker reads each div's BEM class, looks it up in the DB (`slot_synonyms.standalone_block`), emits that block, recurses. **Exactly three exceptions:** atomic-tag swap / chrome-skip at top level / **top-level container wrap (`sgs/container` per FR-22-4 unconditional)**. Per-block behaviour comes from DB rows, not code branches and not new bespoke blocks. The framework stays at ~70 reusable primitives across all clients; mockup-section variations get assembled at runtime from those same primitives.

## The exact bug (Round 2 found it)

**File:** `plugins/sgs-blocks/scripts/orchestrator/converter_v2/convert.py`
**Lines:** 1559-1561
**The bug:**

```python
slug = db.resolve_slug_from_bem(sgs_classes)   # line 1557 — returns None for unregistered slugs
...
if slug is None:                                # line 1559
    return walk_passthrough(node, css_rules, depth, variation_buf)   # line 1561 — RETURNS HERE
...
# Lines 1583-1584 (UNREACHABLE when slug is None):
if is_top_level and slug != "sgs/container":
    return db.emit_sgs_container_wrapping(slug, attrs, children_markup, css)
```

When a `<section class="sgs-brand">` (or any section whose BEM root has no DB-resolved block) reaches the walker as top-level, `db.resolve_slug_from_bem(["sgs-brand"])` returns `None` (no such block registered). Line 1561 returns `walk_passthrough` output. The FR-22-3 exception #3 / FR-22-4 top-level container wrap at line 1583 NEVER fires, because it's gated on `slug != "sgs/container"` and that branch is only reachable when `slug is not None`.

**Empirical proof (Agent 7 cross-cutting audit):** across all 9 boundaries' `convert-trace-b*.jsonl`, the `section_wrap` event fired ONCE (b2 / hero). `walker_branch_taken` fired 34 times, ALL with `branch=atomic_tag_swap`. The other 6 body sections + header + footer all emitted unwrapped passthrough markup. Hero is the only section that passes the wrap check because `sgs/hero` IS a registered block (resolver returns `sgs/hero` → slug ≠ container → exception #3 fires correctly).

The orchestrator's `ensure_root_section_class` then post-injects `"className":"sgs-X"` onto the FIRST emitted child block — **masking the fact that exception #3 never fired**. The result looks superficially rooted but has no `sgs/container` wrapper, no container attrs, no FR-22-4 base, and 0 CSS rules routed to a recipient. That explains the 49-per-section identical "no value extracted" leftover-bucket entries, the `slug: None` per_section_results, the 80-99% pixel-diff.

## The corrected fix shape (4 fixes, all R-22-9 compliant, NO new blocks)

### Fix 1 (W) — Walker ordering fix at convert.py:1559

Move the FR-22-3 exception #3 / FR-22-4 container-wrap check **above** the `slug is None → walk_passthrough` early return. When `is_top_level=True` AND `slug is None`, walk children at `is_top_level=False`, collect them, and wrap in `sgs/container` preserving the section's BEM className. Section CSS lifts to the container's attrs via existing FR-22-5 D1 routing.

Sketch:
```python
if slug is None:
    if is_top_level:
        # FR-22-4: unconditional top-level container wrap
        children_markup = [walk(c, css_rules, variation_buf, depth+1, is_top_level=False)
                           for c in node.children]
        children_markup = [m for m in children_markup if m]
        css = collect_css_for_classes(classes, css_rules)
        return db.emit_sgs_container_wrapping(
            "sgs/container",
            lift_root_supports({"className": " ".join(classes)}, node, css_rules),
            children_markup,
            css
        )
    return walk_passthrough(node, css_rules, depth, variation_buf)
```

**This is NOT a 4th exception.** It makes existing exception #3 (FR-22-4 unconditional top-level container wrap) actually fire for the unresolved-slug top-level case. R-22-3 PASS test stays intact (still exactly 3 named exceptions).

**Spec invokes:** FR-22-3 #3 + FR-22-4 + R-22-3 + R-22-9.

**Predicted impact:** addresses 6 of 7 body sections universally. Per-cell predicted drop from 70-99% to ~25-40% (sections get their `sgs/container` wrapper + lifted section-level CSS; remaining gap = internal BEM elements still need DB rows).

### Fix 3 (S) — `variation_css` `::` serialiser bug

**File:** `convert.py`. Bug introduced at **line 352** in `parse_css`:

Leaked verbatim at **line 417** in `collect_css_for_classes`:

**However — Agent 5's finding conflicts with Agent 7's:**

The `::` malformation is **debug-artefact only, inert downstream, NO pixel-diff impact**.
- **Agent 7 verdict:** The `::` IS cross-cutting in `extract.json::per_section_results[*].variation_css` (66 broken @media rules across 7 body sections); claims 5-15pp impact per cell.

**Resolution needed before dispatching Fix 3:** grep the deployed `variation-d0-d2.css` directly for non-pseudo `::` patterns. Agent 5 already did this — confirmed only `::before` / `::after` pseudo-elements present, no `@media (...) :: .selector { ... }` patterns. **Agent 5 is correct; Fix 3 is debug-artefact correctness only, NOT a pixel-diff lever.**

Still worth fixing for debug-artefact integrity (3-line patch at line 417 splitting on ` :: ` and re-wrapping in `@media (...) { selector { ... } }`), but **do not expect pixel-diff impact**.

**Reclassification:** Fix 3 is LOW priority (debug parity, not closure leverage).

### Fix 4 (F) — Hero render.php InnerBlocks migration (Phase 2 Commit 2.1)

Hero block exists; walker emits InnerBlocks correctly; render.php at lines 706-772 builds `$content_html` from hardcoded scalar attrs (`$headline`, `$sub_headline`, `$cta_primary_text`, `$split_image` etc.) — empty/null because walker put strings in child InnerBlocks per FR-22-3.

**The migration shape (per Agent 4 detail):**

## The "missing" 5th item (deferred)

Agent 6 (ingredients-section) flagged a real C-class gap: **responsive grid suffixes**. `property_suffixes` has only base `Columns → grid-template-columns` (one row). Responsive variants (`columnsTablet`/`Mobile`/`Desktop` per breakpoint) need a `breakpoint` column on `property_suffixes` + walker-side @media breakpoint detection in `collect_css_for_classes`.

**Decision:** defer until after Fix 1 + Fix 2 land and measurement reveals whether this C-class gap is real or whether existing responsive-attr machinery covers it through another path. Per R-22-7 (council fix-shapes are hypotheses, not specs) and R-22-4 (single-variable-change discipline). If post-Fix-1+2 measurement still shows 768 mismatch dominating for sections with `@media`-keyed grid rules, then ship the responsive property_suffixes change as Fix 5.
