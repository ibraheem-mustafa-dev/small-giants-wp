# Adversarial council — Spec 36 synthesis

## Convergent headline
The **engine is sound and platform-verified** (the WP-Platform critic confirmed against live code: the
`wp_navigation` walk-and-emit path already ships, `--sgs-header-height` exists, `<dialog showModal>`
top-layer transform-escape is real, the SGS interactivity pattern uses the PUBLIC API not core-nav private
internals — the §15.7 trap is avoided, and the CPT machinery reuse is plausible). Every critic affirms the
a11y disclosure/dialog split, the real-position mega fix, the converter-emit, and the nav→header decoupling.
**No fatal fiction.**

But the panel converges on **three things the spec under-builds, and one class of error only live-source
review could catch:**
1. **The whole non-coder PRODUCT — how an operator assigns + edits a mega panel — is unspecified** (3 voices).
2. **The reference-integrity / orphan lifecycle is absent** — 4 stores, no one owning cleanup (2 voices) —
   *literally the fragility that rotted the predecessor, with an extra store.*
3. **There is no MVP / phasing / prove-first gate** — the motivating feature (Indus mega) is padlocked
   behind the tedious middle (2 voices).
4. **Three "(exists)" overstatements** — I cited a header-state attribute, two shared controls, and a
   converter as *existing* when they're unbuilt designs or wrong names (WP-Platform, source-verified).

## GO / NO-GO
**CONDITIONAL GO.** The architecture stands — the fixes are additions + corrections + phasing, not a
redesign. Do NOT start building until the convergent must-fixes below are closed on paper. The Cynic's
NO-GO is about the same items (integrity + association mechanism + phasing), which the register closes.
