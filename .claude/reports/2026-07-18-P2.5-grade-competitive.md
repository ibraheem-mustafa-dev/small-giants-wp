# Spec 36 v1.2 — Competitive + Operator-Experience Grade

Grading the CURRENT v1.2 (folded fixes NOT re-reported). Anti-inflation anchors held:
5 = category-leading · 4 = ship today · 3 = competitive w/ visible gaps · 2 = behind the field.

---

## Criterion 1 — Competitive strength (does it beat the field or just match?) → **4.0**

**Evidence.**
- **Genuine exceeds (quality axis):** FR-36-4 ships hover-intent (300 ms attr) + **safe-triangle** + close-grace + WCAG 1.4.13. The inventory's own S1 finding: "WP builders under-build exactly this… the single biggest quality gap vs competitors." Kadence/Spectra megas visibly jank on the diagonal — this beats them.
- **A11y beats the field:** FR-36-10 hard-splits disclosure (`<button aria-expanded>`, no `role=menu`, no trap) vs dialog (native `<dialog showModal>`, top-layer). Most WP nav a11y is broken (role=menu misuse, transform-ancestor focus bugs). This is ahead, primary-source-grounded (APG/MDN/Roselli).
- **Crawlable rich mega, no AJAX (FR-36-17):** server-rendered mega content is a real SEO edge — many builders lazy-load mega panels. The spec scopes the moat honestly ("plain `<a>` is table stakes; the differentiator is server-rendered *rich* mega + AI auto-gen").
- **Matches, not beats (feature surface):** mega-as-CPT + native-menu association is the **Spectra/Astra approach** (spec says so — "restores the early Spectra-style decision"). Off-canvas drawer, sticky/transparent header (the header is Spec 17, out of scope here), per-device visibility — all table-stakes.
- **Behind on breadth:** WooCommerce nav (mini-cart flyout, category mega), multilingual, conditional/role items, command-palette **all deferred to Phase 3** (§7). Kadence Pro / Spectra Pro ship WC + conditional nav today. For an e-comm buyer that is a visible parity gap.

**Reasoning.** It clearly beats the field on *correctness and interaction quality* and matches-or-trails on *visible feature breadth*. The exceeders that actually ship (intent timing, a11y) are quality dimensions; several evidence-backed exceeders (bottom-tab-bar, command palette, WC nav) are dropped or deferred. Solidly ship-to-client, not category-leading on the demo-visible surface. **4.0.**

## Criterion 2 — Operator / non-coder UX (first menu + attaching a mega in ~10 min) → **4.0**

**Evidence.**
- FR-36-1 resolves to the **site's primary menu by default** — the operator uses the menu they already know; no new mental model for the common case.
- FR-36-5 attaches a mega **"the native WordPress way"** — add the mega CPT to the menu like a page. This dissolves the bespoke item→mega map + minted-ID fragility. A non-coder already knows "add item to menu."
- FR-36-14: **`<ServerSideRender>` editor preview** (kills the drift lesson) + `templateLock:"contentOnly"` client-safe editing + "which menu is bound (bar vs drawer)" inspector indicator. All reduce operator confusion.
- **Risk on the headline flow (⚠BUILD-SPIKE, FR-36-5):** the *entire* "native, familiar" UX rests on the **unverified** assumption that the block Navigation editor surfaces a CPT in link search. Fallback = classic-menu support / bespoke "attach panel" affordance — which *degrades the exact pitch* the UX sells. This is load-bearing and unresolved.
- **Assembly surface for the drawer:** attaching a burger→drawer needs a separate `sgs/nav-drawer` block + `drawerRef` wiring. Defaults soften it (unset → single drawer on page; dangling → editor Notice, FR-36-9a), but it's the fiddly bit for a non-coder.
- Per-device visibility depends on **`ResponsiveTriStateControl` (DESIGNED-not-built)** — a control the operator needs but that doesn't exist yet.

**Reasoning.** The common case (Mama's flat bar + burger + drawer) is genuinely ~10-min simple, and the native mega association is an elegant, low-friction bet. Held below 5 by (a) the unverified build-spike under the headline UX, and (b) the CPT + block + drawer + refs the operator assembles for the rich case. **4.0.**

## Criterion 3 — Differentiator durability (real moat vs copyable) → **3.5**

**Evidence.**
- Safe-triangle / intent timing / height-animation — **vanilla JS, trivially copyable**. Low moat.
- A11y correctness — copyable; competitors are slowly closing it.
- Native mega association — *is literally Spectra's existing pattern*. No moat.
- Server-rendered no-AJAX rich mega — **moderately durable**: lazy-loading competitors would have to unwind an architecture to match. Some stickiness.
- **The real moat (FR-36-15 / FR-36-17):** the nav is **emittable-by-construction** by the SGS cloning pipeline — AI auto-generates the whole nav (incl. mega) from a scraped draft. **No WP builder does this.** High durability — but it's a *platform* property this spec *enables*, not a nav feature it *owns*. The spec's contribution is locking the nav into that moat (ID/position-ref, no slug, real-position render, per-client scoping).

**Reasoning.** Within the nav itself the differentiators are copyable. The durable moat is inherited from the pipeline; the spec earns credit for making the nav a first-class emit target rather than an afterthought. "Competitive with a visible durability gap." **3.5.**

## Criterion 4 — Demo-strength (would it win a live demo?) → **3.5**

**Evidence.**
- **Wins a side-by-side:** an SGS mega opening with safe-triangle (no runaway) + backdrop blur + height-animation, next to a janky Kadence mega, is a visible win — but this is **Phase 2**.
- **The showstopper is inherited + deferred:** "watch AI build the nav from a scraped site" is the pipeline, and the mega auto-gen lands Phase 2 (Phase 1 emits only bar+drawer).
- **Invisible in a demo:** a11y (FR-36-10/11) and crawlability (FR-36-17) — the actual quality leadership — don't show on screen unless deliberately surfaced (View Source / axe). Weak demo material as specced.
- **Phase-1 MVP is unremarkable:** flat bar + burger + full-screen drawer = every builder. No wow in the first shippable slice.
- **Lost demo-winners:** command palette (⌘K) and bottom-tab-bar (evidence-backed, competitors rarely offer) are deferred/dropped.

**Reasoning.** The flashy, demo-winning material is either deferred (mega polish, AI-gen mega) or invisible (a11y, crawl). The first shippable slice doesn't wow. Real demo upside exists but isn't sequenced or surfaced to fire early. **3.5.**

## Overall → **3.8 / 5**

A disciplined, honest, ship-ready spec that **leads the field on correctness and interaction quality** and **matches-or-trails on visible feature breadth and durable moat**. It would go to a paying client today (Mama's + Indus). It is not category-leading because: the exceeders that ship are invisible-quality; the durable moat is inherited (pipeline); and several evidence-backed differentiators (bottom-tab-bar, command palette, WC nav, context-aware theming) are dropped or deferred. The spec's self-honesty ("scope the moat honestly," named arbitration steps, ⚠VERIFY flags) is a strength that keeps it from over-claiming.

## Upgrade opportunities — what makes this a SHOWPIECE / category-leader

Non-obvious, showpiece-oriented. All **PLAUSIBLE** (depend on cross-spec/pipeline cooperation) unless noted.

### O1 — "Watch AI build your nav" as the demo centrepiece · showpiece **HIGH** · PLAUSIBLE
The real moat (AI auto-gen from a scraped draft) is buried as an emit contract. **Sequence and stage it as a live demo flow:** paste a competitor's URL → SGS scrapes → auto-builds the full nav *including a rich mega*, live, in front of the buyer. No WP builder does this. Requires the mega to auto-generate early enough to demo (currently Phase 2) + the pack factory (G4). This converts an inherited platform property into *the* category-leader moment. **Directly lifts Criterion 3 + 4.**

### O2 — Context-aware, scroll-driven "signature" nav · showpiece **HIGH** · PLAUSIBLE
The visual teardown's "steal #3" (Shopify: transparent bar, text colour auto-derived from the section behind) is a NICE in the inventory and **absent** from the spec. Combined with declarative scroll-driven hide/shrink, this is the single most *expensive-looking* nav behaviour and almost no WP builder ships it well. Make it a first-class, token-driven, reduced-motion-gated default. Needs header cooperation (transparent mode + section-colour detection is partly Spec 17) and a vanilla-JS scroll baseline (browser-support caveats). Every demo would *look* category-leading, not just measure well.

### O3 — Operator-facing "Nav Health" panel — turn the invisible moat visible · showpiece **MED-HIGH** · PLAUSIBLE (checks mostly already exist)
The spec's strongest differentiators (a11y correctness, crawlability, real-position mega) are **invisible in a demo** — my Criterion-4 penalty. The §8 QC gate already computes them (axe=0, crawl assertion, real-position, integrity sweep) and FR-36-12 already has informational notices. **Surface them in-editor as a live "Nav Health" panel:** green ticks for "every link crawlable by Google + AI search," "keyboard-navigable," "contrast AA," "mega renders at correct position." For a non-coder buyer this converts invisible quality into *visible* demo proof and a trust close — reusing checks that mostly exist. Lowest build cost of the three; directly answers the demo-strength weakness.
