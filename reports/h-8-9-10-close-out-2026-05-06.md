## 5. H-10a — css-pattern-audit.js Extension

Added `checkBackgroundShorthand()` function (~85 lines) between `auditFileFast()` and `auditFile()`. Function:
- Walks CSS line-by-line matching `background:\s*(linear-gradient|radial-gradient|url\s*\()` regex
- Back-tracks up to 20 lines to find the enclosing selector
- Classifies selector as inner-element (has `__`, `::before`, `::after`, or `--skeleton`/`--shimmer`/`--overlay`/`--fade`/`--caption`) vs block-wrapper
- Emits `CRITICAL` / `R4-background-shorthand-missing-has-background-guard` for block-wrapper rules without `:not(.has-background)` — exits code 1
- Emits `WARNING` for rules that have the guard but still use `background:` shorthand (recommend `background-image:`)
- Emits `WARNING` for inner-element shorthand (no guard needed, but longhand conversion recommended)
- Integrated into `auditFile()` alongside existing M1 animation checks

## 6. H-10b/10c — mockup-parity-validator.js Extensions

### H-10b — Pseudo-element measurement
Added to `captureForSelector()` page.evaluate function (after the main WATCHED set collection):
- Calls `getComputedStyle(chosen, '::before')` — if `content !== 'none'`, captures 10 properties (content, display, backgroundColor, backgroundImage, backgroundSize, backgroundPosition, color, opacity, filter, mixBlendMode) into `pseudo_before`
- Same for `::after` → `pseudo_after`
- In `buildJsonReport()`, each delta is enriched with `sgs_pseudo_before`/`sgs_pseudo_after` and `mockup_pseudo_before`/`mockup_pseudo_after` when present

### H-10c — Parent-chain filter walker
Added to `captureForSelector()` after pseudo-element measurement:
- Walks `chosen.parentElement` → `document.body` checking `filter`, `mixBlendMode`, `backdropFilter`, `opacity`
- Any non-default value → appended to `parentChainEffects` array as `"tag.class: filter=hue-rotate(60deg)"` format
- Surfaced as `out.parent_chain_effects` array
- In `buildJsonReport()`, each delta with `s.parent_chain_effects` gains `parent_chain_effect` field and is automatically flagged `requires_screenshot_review: true`

**Test fixture (documented in validator header):** Place `filter: hue-rotate(60deg)` on a parent element over a known-coloured child. Validator should output `parent_chain_effects: ["div.parent: filter=hue-rotate(60deg)"]` on any delta for that child.
