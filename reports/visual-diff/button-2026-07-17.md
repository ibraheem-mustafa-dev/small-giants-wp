# Visual-diff — sgs/button `white-space: nowrap` -> `normal` (base rule, all sites)

**Change:** `plugins/sgs-blocks/src/blocks/button/style.css:35` — the shared `.sgs-button` base
rule's `white-space: nowrap` changed to `white-space: normal`. This is a generic/robust default:
a button now wraps its label onto a second line only when it genuinely does not fit the
available width, instead of forcing a fixed-width overflow. No client value hardcoded; applies
to every SGS site.

**Root cause superseded:** a prior per-site patch (`sites/indus-foods/theme-snapshot.json`)
added a mobile-only `white-space:normal` override inside a `@media (max-width:1023px)` rule.

(lightsalmon-tarsier-683012.hostingersite.com) never uses `white-space:nowrap` on its hero
buttons at ANY breakpoint (confirmed live via `getComputedStyle` at 375/768/1024/1440 before
this change). The per-site override is now redundant for `white-space` and was removed,
leaving only the mobile padding step-down (`padding:15px 20px` at `<=1023px`, matching the
original's measured mobile button padding).
