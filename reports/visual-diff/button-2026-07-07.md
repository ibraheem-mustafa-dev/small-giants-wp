## Change under review (D288)

The `<a>`/`<button>` IS the block root now — the intermediate `.sgs-button-wrapper` `<div>`
is removed so the button is the DIRECT flex child of `sgs/multi-button`. Also: colour controls
switched from plain text-boxes to `DesignTokenPicker` (linked mode), and the editor preview now
resolves token slugs so preset Apply visibly restyles. version 1.3.0 → 1.4.0.

- **render.php** — wrapper attrs (`get_block_wrapper_attributes`) merge onto the `<a>`/`<button>`
  itself (id + class + style + data-preset); the `<div>…</div>` is gone. All scoped selectors
  reconciled `#uid .sgs-button` → `#uid.sgs-button` (id+class on the same element, specificity
  (1,1,0) preserved); icon descendants (`… .sgs-button__icon`) keep their space. Full-width is now
  the `sgs-button--full` modifier on the button.
- **style.css** — `.sgs-button-wrapper--full` → `.sgs-button--full` (single rule; the redundant
  `.sgs-button-wrapper--full .sgs-button{width:100%}` dropped). No `!important`, no animation/first-paint rule.
- **edit.js** — `useBlockProps` applied to the button element (editor/frontend parity, no wrapper
  div); 6 colour `TextControl`s + icon + shadow colours → `DesignTokenPicker linked`; previewStyle
  uses `resolveColorToken` (slug → palette colour) + border-width.
- **block.json** — version bump only.
