### Filter improvements added this session (`scripts/mockup-parity-validator.js`)

1. **Visibility-aware selector matching** — walks `querySelectorAll` and picks first element with no `display:none` ancestor + non-zero rect. Eliminates the `.hero-mobile`/`.hero-desktop` viewport-variant trap.
2. **fontFamily fallback-stack equivalence** — `Inter, sans-serif` and `Inter, system-ui, -apple-system, sans-serif` declared equivalent if primary family matches AND loaded on both sides via `document.fonts`.
3. **CSS keyword equivalents** — `textAlign: start ↔ left`, `minWidth/Height: 0px ↔ auto`, `maxWidth/Height: none ↔ none`.
4. **Font-loaded semantics fixed** — only `'error'` status fails the assertion; `'unloaded'` (lazy weight variants) and `'loading'` (mid-flight) no longer count as failures.
