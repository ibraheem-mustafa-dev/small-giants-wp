## Change under review

Converge `sgs/media`'s two non-interoperating styling systems into ONE (native attrs = single
source), fixing 9 audited defects + adding a CSS `height` (fill) attr. render.php + style.css +
edit.js + block.json changed; version 1.5.0 → 1.7.0.

- **render.php** — new `sgs_media_css_length()` accepts unit-embedded strings (`440px`/`100%`) so
  the converter's values stop being dropped by the old `is_numeric()` guard (D1); border-radius
  resolved per-corner → custom `borderRadius` → native `style.border.radius`, emitted inline on the
  `<img>` in naked + figure modes (D2, STOP-44); device-tier breakpoints `599 → 767` (D3);
  scoped-style selector `#uid.sgs-media__img, #uid .sgs-media__img, #uid .sgs-media__video`
  (self-or-descendant) so naked-mode dimensions land (D6); naked img carries `wrapper_styles`
  (alignment) (D6); object-fit/object-position emitted inline only when non-default (D7); NEW
  `height`/`heightTablet`/`heightMobile` CSS-fill emitted alongside max-height.
- **style.css** — overridable `:where(.sgs-media__img){object-fit:cover}` fallback (D7);
  iframe `aspect-ratio:16/9` moved to `:where(...)` so an explicit ratio wins (D9). No `!important`,
  no animation/first-paint rule added.
- **block.json** — opted media OUT of the shared image-controls extension (removed
  `supports.sgs.imageControls`; other blocks untouched); `__experimentalBorder.radius:false` (single
  radius UI); NEW `height`/`heightUnit`/`heightMobile`/`heightTablet` attrs.
- **edit.js** — "Media Styling" + "Caption & Link" inspector panels + editable alt text + Height (fill) control.
- DB: `sgs-update --stage 1` reseed so css `height` routes to the new `height` attr
  (`attr_for_layer_property('sgs/media','OUTER','height')` → `height`; was `None`). Data-only, R-31-1.
