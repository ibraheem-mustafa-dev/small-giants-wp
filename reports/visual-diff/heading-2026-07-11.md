# sgs/heading — text-wrap fidelity: cloned headings greedy, authored keep balance (D305)

**Verdict: PASS.** Bean's chosen Option 1 for the hero H1 wrap divergence.

**Root cause (proven live):** the theme applies `text-wrap: balance` to ALL headings
(`core-blocks-critical.css` `h1..h6,.wp-block-heading`) — a deliberate enhancement for
AUTHORED content. `balance` re-distributes words for even line lengths, so a cloned
heading wrapped "Made for" instead of the draft's greedy "Made for the" (the draft has NO
`text-wrap` → the CSS-initial `wrap`). Confirmed: draft h1 computes `text-wrap: wrap`, the
clone `balance`; forcing `wrap` on the live clone reproduced the draft's exact wrap.

**Fix (Option 1 — faithful for clones, keep balance for authored):**
- `block.json`: new `textWrap` attr (string, default `''`).
- `render.php`: emits `text-wrap:<value>` at `$root_sel` (`.uid.wp-block-sgs-heading`,
  0,2,0 — beats the theme's `h1..h6` at 0,0,1) when `textWrap` is set (enum-guarded).
- `converter/services/assembly.py` step 3a3: sets `textWrap` to the DRAFT's effective
  text-wrap (declared value, else CSS-initial `wrap`) on cloned headings. DB-gated on the
  block declaring `textWrap` (`db_lookup.block_attrs` — only sgs/heading; inert elsewhere),
  universal, no slug literal (mirrors the step-3a2 tag-identity / step-3b layout-trigger
  precedent; Spec 31 FR-31-5.1 absent→initial). Seeded to the DB via `/sgs-update --stage 1`.
- An AUTHORED heading leaves `textWrap` empty → inherits the theme's `balance`.

## Evidence (live, sandybrown page 1356, post deploy + OPcache reset)
- Converter emits on the REAL draft hero H1:
  `<!-- wp:sgs/heading {"content":"Made for the mum<br>who needs it most","level":"h1","textWrap":"wrap"} /-->`
- Render: a heading with `textWrap:"wrap"` computes **`text-wrap: wrap`** (greedy); a
  heading with no `textWrap` computes **`text-wrap: balance`** (theme enhancement kept).
