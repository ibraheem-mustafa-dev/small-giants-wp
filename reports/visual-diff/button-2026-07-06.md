## Change under review

Full-width (`widthType='full'`) fix — a full-width button did not fill its container when it
was a FLEX ITEM (inside sgs/multi-button's flex row or an sgs/container flex layout): the
wrapper `.sgs-button-wrapper--full` was shrunk by the browser default `flex-shrink:1`, so
`width:100%` never filled the line. Plus two cleanups: the dead `.sgs-button--full` element
class (emitted but styled nowhere), and the editor-vs-frontend divergence (editor forced an
inline `width:100%` on the preview + never added `--full` to the preview wrapper).

- **style.css:** `.sgs-button-wrapper--full` now `width:100%; flex:0 0 100%; max-width:100%;
  align-self:stretch;` — holds the full line as a flex item (basis 100%, no grow/shrink), fills
  the cross-axis in a flex column, and is a no-op in non-flex parents (where `width:100%` already
  worked). Inner `.sgs-button` keeps `width:100%`.
- **render.php:** removed the dead `sgs-button--full` element-class emission (kept the id-scoped
  `#uid .sgs-button{width:100%}` `<style>` + the `sgs-button-wrapper--full` wrapper class).
- **edit.js:** removed the dead `sgs-button--full` push + the inline `previewStyle.width='100%'`
  hack; the editor preview wrapper now conditionally carries `sgs-button-wrapper--full`, so the
  editor exercises the SAME wrapper CSS as the frontend (divergence gone).
- **block.json:** version 1.2.0 → **1.2.1**.

Preset/colour/typography logic (the `$is_custom`/`inheritStyle` gate) deliberately untouched —
that is the separate preset-as-seed task.

## Evidence (live homepage / page 8, sandybrown, deployed 2026-07-06)

Playwright, applying `--full` non-destructively to a real button wrapper inside a live
`display:flex; flex-wrap:wrap` multi-button row (parent width 350px):
- **BEFORE** `--full`: wrapper `199px` (content-shrunk — the bug).
- **AFTER** `--full`: wrapper `350px` = full line width (`fillsLine: true`); inner button expands
  to fill the wrapper's content box.

Confirms a full-width button now fills its container as a flex item, the exact case that failed.
