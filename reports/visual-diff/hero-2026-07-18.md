# Visual-diff — `sgs/hero` inline-zero Facet B (GOTCHA-F CTA hovers) — 2026-07-18

Framework inline-zero rollout (Spec 32 FR-32-4 as amended D345), **Facet B** —
the shared `SGS_Container_Wrapper` now emits a composite's per-instance `--var`
values as a scoped `.{uid}{…}` rule instead of inline. hero renders through the
wrapper, so its `--sgs-cta-*-hover-*` / `--sgs-hover-*` vars moved scoped and the
`.sgs-hero[style*="--sgs-…"]` presence-selectors that gated its CTA/root hovers
had to be rewritten (GOTCHA F). Converted by a `wp-sgs-developer` agent; reviewed.

## What changed (`src/blocks/hero/style.css`)

All 9 `[style*="--sgs-…"]` presence-selectors (6 CTA variants + 3 root-hover)
rewritten: the `[style*=…]` attribute gate removed, each gated declaration now
consumes `var(--the-var, <resting-value>)` with the element's own resting value
as an inert fallback (per-variant: primary→`primary-dark`, outline→`primary`,
accent→`accent`, root→transparent/inherit), so hover applies only when a colour
is set and is a no-op otherwise. No hover VALUES changed. render.php/block.json
untouched.

**Disclosed deviation (open item):** the original CTA rules also forced
`box-shadow:none`/`filter:none` when a custom hover colour was set (to stop the
accent ring / custom filter clashing). That can't be made conditional via `var()`
without a render.php boolean, so it is dropped — a minor cosmetic layering change
ONLY when an operator sets a *custom* hover colour on the accent/custom CTA
variants. The colour-swap (the GOTCHA-F fix) works. Tracked for a 1-line render.php
follow-up if wanted.
