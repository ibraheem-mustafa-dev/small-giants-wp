4 bugs were reported. Investigation found:

- **Bug C (mobile button gap):** Real defect. Fixed.
- **Bug D (desktop full-bleed):** Real defect. Fixed.

## Bug C — Mobile button gap

**Root cause:** The SGS multi-button block renders as `.sgs-hero__ctas` > `.sgs-multi-button.wp-block-sgs-multi-button` (outer flex container) > `.wp-block-sgs-multi-button` (inner block container). The `gap: 10px` on `.sgs-hero__ctas` only applies between direct children — the entire multi-button block is one child. The inner `.wp-block-sgs-multi-button` was `display: block`, so the two button wrappers sat flush with 0px gap between them.

**Fix:** Added to `@media (max-width: 767px)` block in `mamas-munches.json` styles.css:
```css
.sgs-hero__ctas .wp-block-sgs-multi-button .wp-block-sgs-multi-button { display: flex !important; flex-direction: column !important; gap: 10px !important; }
```

**Verified:** btn_0 bottom = y(763)+h(56) = 819. btn_1 top = 829. actualGapPx = 10. Matches mockup `gap: 10px` exactly.

## Bug D — Desktop full-bleed

**Root cause:** `section.sgs-hero { margin: 0 -24px }` assumes parent has 24px padding. Actual parent (`.entry-content.wp-block-post-content`) has `padding: 0 16px`, so the negative-margin overshot by 8px on each side. Hero rect was `x: -8, right: 1417` at 1440 viewport — 23px short on the right edge.

**Fix:** Scoped to `.page-id-29` only (variation-level, no framework impact). Two rules:
```css
.page-id-29 .entry-content.wp-block-post-content { padding-left: 0 !important; padding-right: 0 !important; }
.page-id-29 section.sgs-hero { width: 100% !important; margin-left: 0 !important; margin-right: 0 !important; max-width: none !important; }
```

Zeroing the container padding avoids the `100vw` scrollbar gotcha (where `100vw` = `window.innerWidth` = 1440 but `clientWidth` = 1425 due to 15px scrollbar, producing overflow). With padding zeroed and `width: 100%`, the hero naturally fills the full content-area width with no overflow.

**Verified:** Hero rect = `x: 0, right: 1425, w: 1425` at 1440 viewport. The 1425px width = 1440 minus 15px scrollbar — identical constraint in both mockup and SGS. No gap visible on either edge. Hero starts at x=0.
