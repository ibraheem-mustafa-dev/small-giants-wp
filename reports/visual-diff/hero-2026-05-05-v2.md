## What was fixed

### Fix 1 — Buttons inline at desktop (CLOSED)

**Root cause discovered:** The DOM nesting is deeper than the spec anticipated.
Structure: `.sgs-hero__ctas > .sgs-multi-button > .wp-block-sgs-multi-button(display:block) > .sgs-button-wrapper`.
The outer `.sgs-multi-button` already had flex-row via its own inline `<style>` block, but its
only child (`.wp-block-sgs-multi-button`) was `display: block`, stacking both buttons inside it.

**Fix applied:** Added to `mamas-munches.json` `styles.css` + `wp_global_styles` post 7:
```css
@media (min-width: 768px) {
  .sgs-hero__ctas { flex-direction: row; gap: 12px; align-items: center; flex-wrap: wrap; }
  .sgs-hero__ctas .sgs-button-wrapper, .sgs-hero__ctas .wp-block-sgs-multi-button { width: auto !important; }
  .sgs-hero__ctas .sgs-multi-button > .wp-block-sgs-multi-button {
    display: flex !important; flex-direction: row !important; gap: 12px !important;
    align-items: center !important; flex-wrap: wrap !important;
  }
}
```

### Fix 2 — Page title hidden (CLOSED)

Added: `.page-id-29 .wp-block-post-title { display: none; }`
Page title element exists in DOM, rendered display:none at all viewports.

### Fix 3 — Hero padding 0 + min-height 0 (CLOSED)

Added variation CSS: `.page-id-29 section.sgs-hero { padding: 0 !important; min-height: 0 !important; }`
Hero renders at exactly 720px height matching the mockup. No dead pink bands.
Note: block attribute approach skipped — variation CSS path is guaranteed and instance-scoped.

### Fix 4 — Negative-margin full-bleed (DEFERRED)

Documented at `tools/qc-prevention/full-bleed-pattern-replacement.md`.
Remaining visual symptom: hero rect x may be slightly negative at some viewports due to
`margin: 0 -24px` vs parent 8px padding. Framework-level fix required.
