## What changed
`src/blocks/media/style.css`: a linkless image (naked/root mode — the img IS
the block root, D288 single-element pattern) matched NEITHER max-width guard
selector, so the theme's constrained-layout `max-width:1200px` (0,1,0) beat
the block's zero-specificity fallback and a 500px-intrinsic image overflowed
a 375px viewport. New root-form rule `.wp-block-sgs-media.sgs-media__img`
{ min-width:0; max-width:100%; height:auto } — deliberately WITHOUT
`width:100%` (that is the figure/grid-track behaviour; the first deploy
included it and live probes caught naked-mode images ballooning to the
1392px content column — corrected same cycle).
