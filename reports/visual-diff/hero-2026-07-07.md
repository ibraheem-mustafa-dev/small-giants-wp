## Change under review (D288, point D)

Two changes so the cloned hero matches the draft's default vertical alignment:

- **style.css** — `.sgs-hero--align-left .sgs-hero__content` uses `align-items:stretch` (was
  `flex-start`). `flex-start` conflated text-alignment (already handled by `text-align:left`) with
  child cross-axis sizing, shrink-wrapping the CTA multi-button to ~150px. `stretch` lets block
  children fill the content-column width (full-width mobile CTA stack). No `!important`, no
  animation/first-paint rule.
- **block.json** — `verticalAlign` default `"start"` → `""` (blank).
- **class-sgs-container-wrapper.php** (shared) — skips emitting `align-items` on the grid/flex layer
  when `verticalAlign` is blank, so the grid falls back to the browser/CSS default (`stretch` / the
  hero's own style.css `align-items:center`) instead of being forced to `start`.
