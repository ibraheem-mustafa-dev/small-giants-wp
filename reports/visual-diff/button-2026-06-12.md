## What changed (plain English)

The framework's outline/ghost button (`.sgs-button.is-style-outline`) was wrong: its text
colour and border colour were both set to the brand primary, so it rendered pink text with a
border that disappeared against the section. The mockup's ghost button is the standard pattern:
transparent fill, a visible neutral border, dark text. The fix points the outline style at new
`theme.json` `custom.buttonPresets.outline` tokens (text / border / hover-*), with sensible
fallbacks, so the look is correct out of the box AND each client can override it in their own
snapshot without touching block CSS.
