`style.css` only (QC #4, block-side, no shared-file edit):

was MOVED off `.sgs-hero--split` (the outer 2-column grid) onto
`.sgs-hero--split .sgs-hero__content` (the content column only). On the outer grid it
insets BOTH columns — including the media/image column, which the draft leaves flush to

authored value still wins over this default.

padding was a hardcoded default that insets the image; the converter's L4 per-area
extraction does not yet transfer the draft's `.sgs-hero__content` padding to
`contentPadding*` (leftover-buckets reason "no value extracted") — that deeper L4 fix is
tracked separately.
