# Visual-diff — sgs/hero no-inline + 6 per-area box-object families + F3-drain (D295 rollout)

**Verdict: PASS** · **first_paint_capture_passed: true**

## What changed
`sgs/hero` (section-KIND composite) migrated to the no-inline + box-object contract while
KEEPING `SGS_Container_Wrapper` for the section outer (per the D294 principle: section/layout
composites keep the wrapper; only content-KIND go block-private). The 6 per-area SGS-custom
families — `contentPadding` / `mediaPadding` / `imagePadding` / `imageBorderWidth` (base-only) /
`imageBorderRadius` / `contentBandPadding` — migrated flat→object (incl. Tablet/Mobile tiers;
the non-standard tier-infixed `imageBorderRadiusTabletTL` normalised to
`imageBorderRadiusTablet.topLeft`). All 4 WP supports flipped to
`__experimentalSkipSerialization`; hero emits its own section color/border/typography scoped to
`.wp-block-sgs-hero` (NOT via the wrapper's inlining `extra_styles` — the composite caveat; a
double-emit trace confirmed the wrapper only serialises `style.spacing`, so no property lands
twice). Every remaining block-private inline `style="prop:…"` (imageBorderWidth, object-fit/
position, media/content/overlay backgrounds) moved to the scoped `.uid` `<style>`. `$is_split`
gating preserved. Scoped uid is a CLASS (anchor-safe). No version bump.

## F3-drain
The `align-items:center` hardcode (style.css) is drained: render.php now emits a
`verticalAlignment`-driven scoped `.uid{align-items:…}` rule (top→flex-start / center→center /
bottom→flex-end); default `center` renders byte-identical. Hero's entry deleted from
`hardcoded-render-defaults-baseline.json` (F3 debt 10→9). check-hardcoded-render-defaults = 0 net-new.

## Security
The blub-255 panel + the automated security review found an overlay-colour sibling-sanitiser
asymmetry: `$overlay_colour` was pre-sanitised at assignment while sibling colours sanitise at the
concat site — refactored to store raw + sanitise once at the concat site via `sgs_colour_value()`
(matches the media/content/image-border pattern; no double-apply on a resolved `var()`). Blob via
`wp_strip_all_tags`.

## Known residual (tracked, NOT a hero-migration defect — deferred to the universal grid-scoping task)
The hero section root carries ONE inline property: `display:grid` — emitted by the shared
`SGS_Container_Wrapper`'s grid engine (pre-existing; the split-hero is a grid container). Grid-CSS
scoping was deliberately scoped OUT of this migration and the earlier wrapper fix (it affects EVERY
grid composite — card-grid, feature-grid, container grids — and belongs in one universal pass, per
Spec 31 §3.A which treats `display`/`grid-template-*` specially). Also the child `sgs/label` inside
the hero still emits inline (label is unmigrated — Task 3 roster). Neither is introduced by or owned
by this hero migration.
