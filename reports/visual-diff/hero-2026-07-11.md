# sgs/hero — remove hardcoded `.sgs-hero--split` grid gap (LANDED)

**Verdict: PASS** (for the gap divergence). Bean reported the hero H1 wrapped
"Made for the" → "Made for" a word early at 800px vs the draft.

**Investigation (all measured live, draft vs clone at 800px):**
- The H1 typography is byte-identical between draft and clone: 52px, letter-spacing
  −1px, Fraunces (weight 700, line-height 59.8px). "Made for the" renders at 301.7px in
  BOTH; "Made for the mum" at 438.64px in BOTH — fonts render identically.
- **Gap divergence (this fix):** the draft's `.sgs-hero` grid has NO gap (`normal`/0) →
  columns 400/400 at 800px. The clone's hero had a hardcoded **16px** gap
  (`core-blocks... no — hero/style.css`: `@media (max-width:1024px){ .sgs-hero--split {
  gap: var(--wp--preset--spacing--40) } }`) → columns 395/389, content column 5px
  narrower (299 vs 304 inner). D228: a hardcoded default overriding the draft's
  faithfully-absent value. **Removed** → columns now 400/400, content inner 304 = draft,
  content panel + image flush (matches the draft). Any real draft gap is transferred to
  the `gap`/`gapTablet`/`gapMobile` attrs and emitted by `SGS_Container_Wrapper`.

**Evidence (live, sandybrown page 8, post deploy + full CDN cache clear):**
- Hero grid gap: 16px → **0** (draft = 0). Content column inner width: 299 → **304**
  (draft = 304). Content panel and image render flush, matching the draft.
- Buttons (multi-button, fixed same session) sit side-by-side in the wider column.

**Separate cause tracked (NOT this fix):** even at the corrected 304px column, the clone
H1 still wrapped "the" a word early — root-caused to **`text-wrap: balance`** on the
clone's heading (theme `core-blocks-critical.css` applies it to all headings; the draft
uses greedy `text-wrap: wrap`). Proven: forcing `text-wrap: wrap` on the clone reproduced
the draft's "Made for the" / "mum" wrap exactly. This is a framework-wide design decision
(balance is a deliberate theme enhancement) — deferred to a Bean decision, tracked in
parking (`P-HEADING-TEXTWRAP-BALANCE-VS-DRAFT`).
