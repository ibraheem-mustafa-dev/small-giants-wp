verdict: PASS

**Change (Task 2):** a per-badge `fillStyle` (outline|filled) + `fillColour` control. The icon
CSS previously forced `fill:none; stroke:currentColor` on every badge SVG uniformly, so a solid
glyph (the "Loved by Breastfeeding Mums" polygon star) rendered as a hollow outline. Added:
- `style.css` — an ADDITIVE exemption `.sgs-trust-bar__circle--filled svg { fill: var(--sgs-trust-badge-icon-fill, currentColor); stroke: none }`. It fires ONLY on the new `--filled` modifier class; the base outline rule is unchanged.
- `render.php` — emits the `--filled` class + `--sgs-trust-badge-icon-fill` var only when `item.fillStyle === 'filled'`.
- `edit.js` — "Filled icon" toggle + fill-colour picker in the badge inspector; the editor preview mirrors it.

**Why first-paint-safe:** no `@media`, no animation, no transition, no `opacity`/`transform` on
first paint — a static `fill`/`stroke` override on an opt-in modifier class. There is no first-paint
flash surface (the M1 concern this gate guards).

**Validation (live, R-22-11 / R-31-11):** built (`npm run build`, trust-bar style-index.css emitted,
dead-controls guard clean) and deployed to the sandybrown canary (`build-deploy.py --blocks-only`),
OPcache reset. Verified on the live homepage (page 8) via chrome-devtools:

  outline stroke `rgb(197,106,122)`, no `--filled` class. **Zero visual change to existing content.**
