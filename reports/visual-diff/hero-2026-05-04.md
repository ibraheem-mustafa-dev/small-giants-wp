## Deployment notes

The `wp_global_styles` post (post type, ID 7 on this site) caches the merged theme.json + variation. Editing the variation file + cache flush is NOT enough — the post content has to be reset and re-applied. The procedure that worked:

1. Reset post 7 via `POST /wp-json/wp/v2/global-styles/7` with empty `{settings:{},styles:{}}`
2. Re-apply the Mama's variation by reading `/wp-json/wp/v2/global-styles/themes/sgs-theme/variations`, finding the `Mama's Munches` entry, and POSTing its full `settings`+`styles` back to post 7
3. `wp cache flush` + LiteSpeed CSS dir clear
4. OPcache reset via HTTP fetch of a temp PHP file

This pattern should be encoded as a `/sgs-deploy` post-deploy step or cycle into the `deploy` skill for any future style-variation edits.
