## Minor caveat (NON-blocking, flagged for cleanup)
Computed-style check: `.sgs-mega-panel--promo`/`--cards` resolve to `display:flex`
not `grid` — core's `.wp-block-columns{display:flex}` overrides the new single-class
grid rules (specificity tie, core wins on source order). **No visual impact**: the
patterns are built on `core/columns` with explicit 66.66/33.33 widths, so core's
flex produces the same layout the grid classes intended. The two new grid classes
are effectively redundant on these `core/columns`-based patterns. Cleanup options
(later): either boost specificity (`.wp-block-columns.sgs-mega-panel--promo`) so the
SGS classes take effect, or drop them since core columns already provide the layout.
Tracked as a minor follow-up; does not regress the rendered output.
