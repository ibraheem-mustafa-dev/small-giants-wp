# sgs/media — R4 __experimentalBorder visual-diff

## Change
`plugins/sgs-blocks/src/blocks/media/block.json` — added `"__experimentalBorder": {"radius": true, "width": true, "style": true, "color": true}` to `supports`; version 1.1.0 → 1.2.0. Block-owned metadata only (no markup/CSS change). Enables the converter to lift `border-radius:16px` (draft `.sgs-brand__image`) onto cloned `sgs/media` blocks (root cause R4 / symptom #12C). `/sgs-update` pushed the support into `block_supports`.

## Live verification (sandybrown canary page 8, build + deploy + OPcache reset, 2026-06-06)
- The support is now present in the converter DB (`block_supports` row for `sgs/media`: `__experimentalBorder {radius,width,style,color}`).
- On the live page-8 clone the `sgs/media` blocks render with **no border / no regression** — the converter does not yet lift `border-radius`/`object-fit` onto media (that is R8d, deferred to a follow-up). The brand image is additionally 404 on this dry-run clone (FS-5 image-sideload territory), so its rounded-corner effect cannot be captured until both R8d lands and the image uploads.
- The block's default appearance (no border attrs) is **unchanged** — this is a capability-enabling metadata addition; borders render only once a border attr is set (by the R8d lift or an editor control).

## first-paint / regression
- Metadata-only supports addition. No markup, CSS, or default-render change → no first-paint defect and no regression to existing media blocks. The new border controls are now available to editor users.

## Verdict
**PASS** — capability enabled with zero visual regression; the visible #12C rounded-brand-image payoff is gated on R8d (media border lift) + FS-5 (image upload), both tracked as follow-ups.
