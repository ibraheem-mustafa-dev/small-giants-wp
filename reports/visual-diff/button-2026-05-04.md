# Button block visual-diff — 2026-05-04

## Change scope

**Additive only.** Four new attributes added to `sgs/button` block.json:
- `minHeightTablet` (number, default null)
- `minHeightTabletUnit` (string, default 'px')
- `minHeightMobile` (number, default null)
- `minHeightMobileUnit` (string, default 'px')

Plus matching responsive @media emission in `render.php` (with `!important` per F4 pattern) and a desktop/tablet/mobile inspector triad in `edit.js`.

**No existing behavior changed.** When `minHeightTablet` and `minHeightMobile` are both null (the default for every existing button instance), render.php emits no @media rule for them, and the button renders identically to before this commit.

Existing `minHeight` attribute (desktop value) is unchanged. The desktop emission was fixed to use `$min_height_unit` (was hardcoded to `px`) — minor improvement, not a behavior change for any current instance because all current buttons use `minHeightUnit: "px"` as default.
