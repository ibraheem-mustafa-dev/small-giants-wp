# sgs/timeline — no-inline + box-object migration (LANDED)

**Verdict: PASS.** Block-private (has SHADOW support + imageControls; renders a timeline of entries).
Verified LANDED on sandybrown page 1356 at 375/768/1440 via the harness.

## Evidence (harness, all breakpoints)
- **Zero inline** (#1): root `.sgs-timeline` + every entry/time/img carry NO CSS property declaration
  (only permitted `--sgs-connector-colour`/`--sgs-object-position` vars). Clean.
- **Box computed** (#10, asymmetric): padding `5px 17px 9px 23px` → `4px 12px 6px 16px` →
  `2px 8px 4px 10px`; margin T/B `10px`/`6px` → `8px`/`4px` → `6px`/`2px`; border-radius
  `7px 13px 25px 19px` → `3px 6px 12px 9px` → `2px 4px 8px 6px`.

## Migration summary
- Supports skip-serialised: spacing, color, __experimentalBorder (radius), typography, **shadow** → scoped `.uid` CSS.
  `box-shadow` emitted via `wp_style_engine_get_styles` (native shadow support, scoped — NOT a box family).
- Border took the quote route (custom `borderWidth` object + `borderColour`/`borderStyle` scalars; radius native + tiers).
- Box-object attrs: paddingTablet/Mobile, marginTablet/Mobile, borderRadiusTablet/Mobile, borderWidth. `box_family` seeded centrally.
- Typography routed to `.sgs-timeline__title` per the block's declared `selectors.typography` (more correct than the
  pre-migration root auto-inline). view.js only toggles `.is-revealed` — no inline style.
- Device tiers 1023/767. uid = class (anchor-safe). No version bump / deprecations (D293).
