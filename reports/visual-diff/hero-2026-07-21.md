# hero dead-CTA purge — visual verification

## What changed
sgs/hero migrated its CTAs to InnerBlocks children at FR-22-6; render.php states the scalar
CTA attrs are "deliberately NOT read here. R-22-14: no fallback." Fourteen attributes and
the CSS they drove were therefore inert. Removed:

- 6 legacy scalars: ctaPrimary/SecondaryStyle|Text|Url
- 4 hover attrs: ctaPrimary/SecondaryHoverBackground|Colour
- 4 gap attrs: ctaGap, ctaGapMobile, ctaGapTablet, ctaGapUnit
- the CSS rules targeting .sgs-hero__cta-primary / --accent / --primary / .sgs-hero__ctas
- the orphaned D345 Facet-B suppression block and its gating-class emission
- the "Button gap" editor controls

Also fixed a separate defect found during review: HERO_CONTENT_TEMPLATE seeded `content` on
the child sgs/label, but sgs/label declares `text`, not `content`. WP silently discards
undeclared attrs (D338), so inserting a hero produced an EMPTY eyebrow label. Changed to
`text`.

## Evidence the 14 were dead — three independent methods
1. repo-wide grep: the 6 scalars appear ONLY in block.json, nowhere else in src/ or includes/
2. LIVE DOM on the canary homepage: every selector the removed CSS targeted renders ZERO
   times — `.sgs-hero__cta-primary` 0, `.sgs-hero__cta--accent` 0, `.sgs-hero__cta--primary` 0,
   `.sgs-hero__ctas` 0. Hero CTAs are `sgs/multi-button > sgs/button` children (1 multi-button
   present).
3. hero has NO shared renderer in includes/ — render.php is the only render path, so the
   live-DOM zero-count is conclusive.

## What was deliberately KEPT (and why the first analysis was wrong to doubt it)
`headlineMarginBottom` / `headlineMarginBottomMobile` target `.sgs-hero__headline`. That class
IS emitted — not by hero's own markup, but by HERO_CONTENT_TEMPLATE applying
`className: "sgs-hero__headline"` to the child sgs/heading (edit.js:132). The canary homepage
lacks it only because that hero was built by the CLONING PIPELINE rather than from the
template. Live DOM is a sample of the instances you look at, not proof about every creation
path. Kept, correctly.

`headline` / `subHeadline` remain DECLARED but unread (render.php:105). Not deleted this pass —
they are referenced by the converter's field extractors and would need their own
stored-content check.

# ADDENDUM — residual .sgs-hero__cta class family removed (same day, second change)

The report above covered the 14-attribute purge. This addendum covers a SECOND change to
hero's stylesheets on the same date. Recorded here because the visual-diff gate keys on
block+date and would otherwise have passed this change on the strength of the earlier
report — a stale pass.

## What changed
The first purge deliberately left a residual family, now removed from style.css:
`.sgs-hero__cta` (base), `.sgs-hero__cta--outline`, `.sgs-hero--split .sgs-hero__cta--outline`,
`.sgs-hero__cta--outline:hover` (x2), `.sgs-hero__cta--custom`, `.sgs-hero__cta--custom:hover`
(x2), the mobile media-query `.sgs-hero__cta` rule, and the obsolete "GOTCHA F rewrite"
comment block. The orphaned custom properties `--sgs-cta-sec-hover-bg` /
`--sgs-cta-sec-hover-colour` went with them (confirmed unreferenced repo-wide and no longer
emitted by render.php).

editor.css: removed `.sgs-hero__ctas .sgs-hero__cta { pointer-events: none }` and
`.sgs-hero__ctas { gap: var(--sgs-hero-cta-gap, …) }`. Both were inert — nothing emits the
singular class, and --sgs-hero-cta-gap stopped being emitted when ctaGap* was deleted.

## CRITICAL DISTINCTION preserved
`.sgs-hero__cta` (SINGULAR) = dead rendered-output styling, removed.
`.sgs-hero__ctas` (PLURAL) = LIVE draft-side vocabulary, PRESERVED. The cloning converter
recognises `.sgs-hero__ctas` in SGS-BEM draft mockups and maps it to sgs/multi-button
(scripts/converter/recognition.py:83; converter/tests/test_container_default.py asserts on
it). Two client stylesheets also reference it (theme/sgs-theme/styles/mamas-munches.css:61,66
and sites/indus-foods/theme-overrides/assets/css/variation-indus-foods.css:154). Verified
still present in all three after this change. Deleting or renaming it would break draft
recognition in the cloning pipeline.
