## Problem
The adaptive-nav drawer programmatically `.focus()`-es its first link on open (intentional
a11y open-focus). theme.json `styles.elements.link` `:focus` (`theme/sgs-theme/theme.json:436-443`)
sets `text-decoration: underline` + `color: primary-dark` and — because theme.json v3 has no
`:focus-visible` key — fires on ANY focus, so the first link shows an underline the instant the
drawer opens (reads as "selected" before any user action).

## Fix (block-scoped, WCAG-preserving)
`plugins/sgs-blocks/src/blocks/nav-menu/style.css`:
```css
.sgs-nav-menu__link:focus:not(:focus-visible),
.sgs-nav-menu__sublink:focus:not(:focus-visible) {
	color: inherit;
	text-decoration: none;
}
```
`:focus:not(:focus-visible)` = focused-but-not-by-keyboard (a programmatic `.focus()` does not
match `:focus-visible` in Chromium — verified by the prior implementer live). REAL keyboard focus
(`:focus-visible`) still gets the theme underline → WCAG 2.4.7 focus visibility preserved.
