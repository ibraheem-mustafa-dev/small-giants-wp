## Problem (container-block hardcode audit #2, Medium)
`plugins/sgs-blocks/src/blocks/mega-menu/style.css:227` — the contained mega-menu panel read
`max-width: var(--wp--style--global--content-size, 1200px)`. The `1200px` fallback is a
hardcode (R-31-1 violation), the same anti-pattern as the hero. Here the fallback value
matches `contentSize` exactly, so it is render-identical while the token is present, but it
must not silently cap at 1200px on a client whose theme.json omits `contentSize`.

## Fix (universal, shared block — no per-site patch)
```css
.sgs-mega-menu--layout-contained .sgs-mega-menu__panel,
.sgs-mega-menu--panel-content .sgs-mega-menu__panel {
	max-width: var(--wp--style--global--content-size);
}
```
Drops the px fallback so the theme.json token is the sole source. Same pattern as the hero fix.
