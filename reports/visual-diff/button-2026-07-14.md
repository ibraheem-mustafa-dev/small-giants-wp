# sgs/button — responsive labelCollapse

New `labelCollapse` enum (`none`|`mobile`|`tablet`|`all`, default **`none` = OFF**) —
a per-instance setting that hides the button's text and collapses it to just the
icon from a chosen breakpoint down. Mirrors sgs/business-info 1:1. The previously
bare label is now wrapped in `.sgs-button__label`; scoped `@media` clip on
`.{uid}.sgs-button .sgs-button__label` (a11y-preserving clip → accessible name
kept, no aria change). Editor: a "Collapse label to icon" SelectControl in the
Icon panel, gated on `hasIcon && iconPosition !== 'only'`.

**Bean's no-icon rule:** the collapse is a NO-OP when the button has no icon/emoji
(else it leaves an empty button). Render gates the clip on `$icon_html` truthy.

(APPROVED — spec + no-icon rule + quality, 0 critical/important). block.json
version 1.5.1→1.5.2. Dead-control + no-inline + all prebuild gates green.
