# Visual diff — sgs/button (D322: hover-transform consumption)

## Change
`button/style.css` now consumes the client `--wp--custom--button-presets--{slot}--hover-transform`
token via a new `--sgs-btn-transform-hover` var, applied in `.sgs-button:hover,:focus-visible`
with a `none` fallback. This closes the FR-33-4 open-bag "transform-lift" contract (the extractor
emits the token; the button now paints it). block.json version 1.5.0 → 1.5.1 (cache-bust).

## Verdict
**PASS — no visual regression.** The `none` fallback preserves current behaviour exactly (Mama's
buttons use colour-invert hovers, no transform); the new capability is inert until a client draft
declares a button hover transform. The per-instance `scaleHover` rule still overrides by specificity;
reduced-motion still forces `transform:none !important`.
