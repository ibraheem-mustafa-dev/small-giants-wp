## What changed (`src/blocks/button/render.php`)

`sgs/button` is block-private: the `<a>`/`<button>` IS the block root (D288), built
via `get_block_wrapper_attributes()`. Its per-instance custom-property VALUES
(`$inline_styles` → `$merged_style`: `--sgs-btn-color/bg/border` + hover variants
+ `--sgs-btn-icon-gap`) were passed as the `'style'` key and rendered inline. They
now emit as a scoped `.{uid}.sgs-button{ … }` rule appended to the block's existing
`$scoped_css_parts` (class-level, D303), and the `'style'` key is dropped from the
wrapper call. The block already has no `[style*="--"]` presence-selectors (GOTCHA F
N/A) and already declared `__experimentalSkipSerialization` per-feature.
style.css's `:hover` rules consume the `--sgs-btn-*-hover` vars via `var()` on the
same element regardless of where the custom properties are declared, so hover
behaviour is byte-identical. No value changed; no version bump; no `deprecated.js`.
